Hur du skaffar och installerar Remote Incident Manager beror på ditt användningsfall och din konfiguration, men processen är i alla fall ganska enkel.
<Windows>
### Typisk installation för enskilda personer
Det finns bara ett installationspaket att använda, oavsett om du erbjuder eller tar emot assistans.
1. [Ladda ner Remote Incident Manager](https://getrim.app/win)
1. Kör den nedladdade filen.
1. Om du har aktiverat användarkontokontroll trycker du på Alt+Y för att godkänna installationen.
1. När filen har startats kommer installationen att påbörjas automatiskt. Det finns inga nästa-knappar att oroa sig för. Luta dig bara tillbaka, slappna av och låt det göra sitt. Inom några sekunder kommer programmet att installeras och startas automatiskt.  
    1. På välkomstskärmen kan du välja hur du i första hand vill använda RIM. Om du bara planerar att ta emot support trycker du på knappen "Få hjälp nu". Om du planerar att ge support klickar du på knappen "Ge hjälp nu". Detta beslut är inte slutgiltigt, eftersom läget alltid kan ändras inifrån programmet.

Det är allt som behövs! En genväg till skrivbordet kommer automatiskt att skapas för att underlätta vid senare start av RIM. Dessutom skapar RIM en global snabbtangent för flera ändamål (Windows+Shift+Backspace) som bland annat automatiskt startar programmet var du än befinner dig.
### Installerare för obevakad eller uppmanad åtkomst
Om den person som ger dig assistans har gett dig ett anpassat installationsprogram kommer du att informeras om detta när installationsprogrammet startas. Detta beror på att installationsprogrammet automatiskt konfigurerar RIM för det som kallas obevakad eller uppmanad åtkomst. Detta gör att den som hjälper dig kan starta en session med din dator utan att du behöver ange ett nyckelord. Vi kommer att gå igenom dessa typer av sessioner mer ingående längre fram i den här handboken.

### Installera utan administratörsbehörighet

Installationsprogrammet för Remote Incident Manager kommer alltid att försöka få administratörsbehörighet. I miljöer där administratörsbehörighet inte kan erhållas kommer installationen av RIM att utföras inom ramen för det aktuella användarkontot, snarare än på systemnivå.

Att köra utan administratör innebär inga begränsningar på styrenhetssidan av RIM. Det finns dock begränsningar som du kommer att stöta på i målmaskinen.

* RIM körs endast när den Windows-användare som installerade det är inloggad. 
* RIM körs inte på inloggningsskärmen i Windows. 
* RIM har ingen åtkomst till det säkra skrivbordet; det innebär ingen åtkomst till UAC-meddelanden eller till inloggningsskärmen när du låser upp maskinen. 
* Fjärrstyrd tangentbordsinmatning fungerar inte om fokus på målet är i ett upphöjt program eller till och med ett program som körs med Windows UI Access-flaggan, till exempel en installerad kopia av en skärmläsare. 
* Alternativet att trycka på Ctrl+Alt+Delete fungerar inte. 

Som controller kommer du att kunna försöka ominstallera RIM med administratörsbehörighet under en session.

### Tyst installation

Du kommer att vilja göra detta när du distribuerar RIM till flera maskiner. Använd helt enkelt kommandoradsparametern **/S** i ditt distributionsskript. Observera att det är skiftlägeskänsligt, så ett stort S krävs.

### Ytterligare överväganden för Windows-servrar

För att installera RIM på din Windows Server, anslut till servern via RDP. För att få den mest stabila anslutningsupplevelsen rekommenderar vi att du använder appen Remote Desktop från Microsoft Store.

#### Hämta ljud och köra Narrator

För Windows Server-distributioner före 2022 måste du aktivera ljudet manuellt innan du startar Narrator. Gör följande när du är ansluten:
1. Tryck på Windows-tangenten+r.
1. Skriv cmd och tryck på enter. Vänta cirka tio sekunder.
1. Skriv in cmd:  
**net start audiosrv**  
och tryck på Enter.

I vilket fall som helst, tryck på CTRL+Windows+Enter för att aktivera Narrator. Om allt fungerade som det skulle borde du få upp tal.  
Du kan nu följa installationsproceduren enligt beskrivningen ovan.
Observera att för Windows Server-distributioner före 2022, om du behöver ljud för efterföljande anslutningar via RIM, måste du installera en virtuell ljudutgångsenhet på din server. Den kostnadsfria versionen av [Virtual Audio Cable] (http://virtualaudiocable.org/download/software/VACSetup.exe) bör räcka.
</Windows>
<MacOS>
### Typisk installation för privatpersoner

Det finns bara ett installationspaket att använda, oavsett om du erbjuder eller tar emot hjälp.
1. [Ladda ner Remote Incident Manager](https://getrim.app/mac)
1. Gå till din nedladdningsmapp via Finder (Alternativ+kommando+L). Leta reda på den nedladdade .pkg-filen och öppna den med Command+O.
1. En installationsguide kommer att visas. Följ instruktionerna på skärmen.  
Nu är du redo att köra Remote Incident Manager för första gången! Du kan köra den från din applikationsmapp, som du kommer åt via Command+Shift+A i Finder. Dessutom skapar RIM en global snabbtangent för flera ändamål (Option+Shift+Backspace) som bland annat automatiskt startar programmet var du än befinner dig.

### Behörigheter

På grund av macOS försiktighetsbeteende när man hanterar applikationer av den här typen för första gången måste du ge vissa behörigheter till Remote Incident Manager vid första lanseringen.

#### Karabiner HID drivrutin

Detta är den drivrutin som bearbetar indata som tas emot av RIM. Det här är ett systemtillägg, så du måste godkänna det manuellt. Rim kommer att visa en systemdialog som tar dig till avsnittet Sekretess och säkerhet i systeminställningarna. Du ska sedan bläddra tills du hittar knappen Tillåt. Om du klickar på den kommer du att uppmanas att ange ditt lösenord eller touch-ID, varefter du kan bli informerad om att du måste starta om för att fullt ut aktivera Karabiner Elements. Om du startar om nu eller senare är upp till dig, men om du bestämmer dig för att inte starta om förrän alla behörigheter har godkänts, lämna Systeminställningar och gå tillbaka till RIM.

#### Tar emot input

RIM behöver tillstånd för att ta emot all input som skickas till den via en fjärranslutning.  
Klicka på "Fortsätt" i RIM-fönstret, varvid en systemdialog visas som begär tillstånd för tillgänglighetstjänster. Om VoiceOver inte fokuserar den omedelbart trycker du snabbt två gånger på **VO+F1**. Tryck sedan på nedåtpilen till "Systemdialoger", tryck på högerpilen och sedan nedåt till begäran om inmatningsbehörigheter. Om du trycker på Enter öppnas varningsrutan, och då ska du följa den till Systeminställningar och markera rutan Remote Incident Manager i listan över program.

#### Fånga skärmutmatning

Nästa begäran om tillstånd från RIM gäller att fånga skärmutmatningen som kommer att skickas över fjärranslutningen. Proceduren för detta liknar den föregående proceduren.

#### Meddelanden

Den sista begäran om tillstånd från RIM gäller aviseringar. RIM kommer att meddela dig när en session startar och avslutas. Proceduren för att bevilja detta tillstånd liknar den föregående proceduren.

### Välkomstskärm!

När alla behörigheter har godkänts kommer välkomstskärmen att visas. Observera att om du väljer att starta om senare, är det nu en bra tidpunkt att göra det. Vid det här laget är RIM redo att initiera eller ta emot anslutningar.

## Avinstallera RIM

Om du stöter på några problem med din installation av RIM, kanske du vill utföra en ren ominstallation av appen. På grund av den komplexa karaktären hos RIM-distributionen för Mac OS kommer appen inte att avinstalleras genom att bara flytta den till papperskorgen, och därmed kanske inte lösa problemet helt och hållet.

RIM tillhandahåller en enkel metod för att avinstallera sig själv helt och hållet. När du klickar på knappen "Om" i huvudfönstret kommer du att se en avinstallationsknapp i skärmen Om som visas. Om du klickar på den här knappen kommer du att få en bekräftelsedialog, varefter du uppmanas att ange ditt lösenord eller touch-ID för att kunna fortsätta avinstallationen. Du kommer att få ett bekräftelsemeddelande när avinstallationen är klar.
</MacOS>
